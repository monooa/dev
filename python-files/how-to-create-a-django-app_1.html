How to create a Django app

aymen March 29, 2025

So, you've successfully set up your Django project using django-admin startproject. You've maybe even run the development server and seen the default rocket ship page. Awesome! But where do you actually write the code for your website's features – the blog posts, the user profiles, the product catalog?

The answer lies in Django Apps.

Think of your Django Project as the main house – it handles the overall configuration, settings, and URL routing. Django Apps, on the other hand, are like the rooms within that house, each dedicated to a specific function or feature. A project can contain multiple apps, promoting modularity and reusability.

For example, a large e-commerce site might have separate apps for:

  • products: Handles product listings, details, categories.
  • orders: Manages shopping carts and checkout processes.
  • users: Deals with user registration, login, and profiles.
  • blog: Contains articles and comments.

This separation makes your codebase cleaner, easier to manage, and allows you potentially reuse an app (like the blog) in another project later.

Ready to build your first room? Let's create a Django app!

Prerequisites

  • Python and pip installed.
  • Django installed (pip install django).
  • A Django project already created (using django-admin startproject myproject).

Step 1: Navigate to Your Project Directory

First things first, open your terminal or command prompt and navigate into the directory where your manage.py file lives. This is crucial because we'll be using manage.py to create the app.

cd path/to/your/myproject/
# You should see the manage.py file if you list the contents (ls or dir)
ls
# Output should include: manage.py myproject/ ... (other files/dirs)

(Replace path/to/your/myproject/ with the actual path to your project)

Step 2: Create the App using startapp

Now, use the manage.py utility with the startapp command, followed by the name you want to give your app. App names should generally be lowercase and descriptive. Let's create an app called polls as an example:

python manage.py startapp polls

(If you're using python3, use python3 manage.py startapp polls)

After running this command, look inside your project directory again. You'll see a new directory named polls has been created!

ls
# Output now includes: manage.py myproject/ polls/ ...

Step 3: Explore the App Structure

Let's peek inside the newly created polls directory. Django automatically generates a basic structure for you:

polls/
├── __init__.py         # Tells Python this directory should be treated as a package.
├── admin.py          # Configuration for the Django admin interface for this app's models.
├── apps.py           # Configuration for the app itself.
├── migrations/       # Directory to store database migration files.
│   └── __init__.py
├── models.py         # Where you define your app's database models (tables).
├── tests.py          # Where you write tests for your app.
└── views.py          # Where you define the logic that handles requests and returns responses (your pages/endpoints).

Each of these files serves a specific purpose in the lifecycle and functionality of your app. We'll focus on views.py, models.py, and potentially creating a urls.py file later as we build out features.

Step 4: Register Your App with the Project

Just creating the app directory isn't enough. You need to tell your main Django Project that this new polls App exists and should be included.

  1. Open the settings.py file located inside your project's configuration directory (e.g., myproject/settings.py).
  2. Find the INSTALLED_APPS list.
  3. Add the name of your app's configuration class to this list. By default, this is usually <app_name>.apps.<AppName>Config. For our polls app, it's polls.apps.PollsConfig. Alternatively, you can often just add the app name ('polls'), and Django can usually figure it out, but using the full config path is more explicit.
# myproject/settings.py

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',

    # My Apps
    'polls.apps.PollsConfig', # Add this line!
    # Or simply: 'polls',
]

Why register? This tells Django to:

  • Include the app's models when running database migrations (makemigrations, migrate).
  • Look for admin configurations within the app's admin.py.
  • Discover templates and static files within the app's directories (with proper configuration).
  • Find tests within the app.

Step 5: Create a Simple View (Optional but Recommended)

Let's make our app do something minimal.

  1. Open polls/views.py.
  2. Add a simple function-based view:
# polls/views.py
from django.http import HttpResponse

def index(request):
    return HttpResponse("Hello, world. You're at the polls index.")

Step 6: Create App-Specific URLs (Optional)

To access this view, we need to map a URL to it. It's best practice to keep app-specific URLs within the app itself.

  1. Create a new file named urls.py inside your polls directory (polls/urls.py).
  2. Add the following code to define a URL pattern for the index view:
# polls/urls.py
from django.urls import path
from . import views # Import views from the current directory

urlpatterns = [
    path('', views.index, name='index'), # Map the root URL of the app to the index view
]

Step 7: Include App URLs in Project URLs (Optional)

Finally, tell the main project's URL configuration about the URLs defined within the polls app.

  1. Open your project's urls.py file (e.g., myproject/urls.py).
  2. Import include from django.urls.
  3. Add a path entry that points a URL prefix (like polls/) to your app's urls.py file.
# myproject/urls.py
from django.contrib import admin
from django.urls import path, include # Make sure include is imported

urlpatterns = [
    path('admin/', admin.site.urls),
    path('polls/', include('polls.urls')), # Add this line
]

Now, if you run the development server (python manage.py runserver) and navigate to http://127.0.0.1:8000/polls/, you should see the "Hello, world..." message from your polls app's index view!

Conclusion

Congratulations! You've successfully created your first Django app, registered it with your project, and even wired up a basic view and URL.

Creating apps is fundamental to building organized, maintainable, and scalable Django web applications. By separating concerns into different apps, you make your project easier to understand, debug, and extend.

From here, you can start defining models in models.py, creating more complex views in views.py, designing templates, and exploring the power of the Django framework. Keep building!
