How to install Django on Windows

[MISSING TRANSLATION] aymen March 29, 2025

So, you want to dive into the world of web development with Django, the high-level Python web framework that encourages rapid development and clean, pragmatic design? Excellent choice! While many tutorials lean towards Linux or macOS, setting up Django on Windows is straightforward once you know the steps.

This guide will walk you through installing Django on your Windows machine, getting you ready to build your first web application.

Prerequisites:

Before we start installing Django, you need Python installed on your system. Django is a Python framework, so this is non-negotiable!

  1. Download Python: Go to the official Python website: https://www.python.org/downloads/windows/
  2. Download the Installer: Grab the latest stable release (usually the big yellow button). Choose the "Windows installer (64-bit)" unless you have a very old 32-bit system.
  3. Run the Installer:
    • Crucial Step: On the first screen of the installer, make sure to check the box that says "Add Python X.X to PATH" (where X.X is the version number). This allows you to run python and pip commands directly from your command prompt.
    • Choose "Install Now" for the recommended installation settings, which usually include pip (Python's package installer) and IDLE (a basic Python editor).
    • Allow the installation to complete.

Step 1: Verify Python and Pip Installation

Let's make sure Python and its package manager, pip, are installed correctly and accessible from the command line.

  1. Open Command Prompt or PowerShell: Search for cmd or PowerShell in the Windows Start menu and open it.
  2. Check Python Version: Type the following command and press Enter:
    python --version
    You should see the Python version you just installed (e.g., Python 3.11.4).

  3. Check Pip Version: Type this command and press Enter:

    pip --version

    You should see the pip version associated with your Python installation.

  4. Troubleshooting: If you get an error like "'python' is not recognized..." it likely means Python wasn't added to your PATH during installation. The easiest fix is often to reinstall Python, ensuring you check the "Add Python to PATH" box this time.

Step 2: Create a Project Directory (Good Practice)

It's good practice to keep your projects organized. Let's create a dedicated folder for your Django work.

  1. Navigate (Optional): You can use the cd (change directory) command in your Command Prompt to navigate where you want to store your projects (e.g., cd Documents).
  2. Create Directory: Make a new folder for your project:
    mkdir mydjangoproject
  3. Enter Directory: Move into the newly created folder:
    cd mydjangoproject

Step 3: Set Up a Virtual Environment

This is a highly recommended step. A virtual environment creates an isolated space for your project's dependencies (like Django). This prevents conflicts between different projects that might require different versions of the same library.

  1. Create the Virtual Environment: While inside your mydjangoproject directory, run:

    python -m venv venv
    • python -m venv: Tells Python to run the built-in venv module.
    • venv: This is the name we're giving to our virtual environment directory (you could name it something else, like .env or env, but venv is common). You'll see a new folder named venv appear in your project directory.

  2. Activate the Virtual Environment: This is the crucial part. You need to activate the environment every time you work on this project.

    .\venv\Scripts\activate
    • Notice the backslashes (\) used in Windows paths.
    • After running this, you should see (venv) appear at the beginning of your command prompt line, indicating the virtual environment is active.

  3. To Deactivate: When you're done working, simply type deactivate and press Enter.

Step 4: Install Django

Now, with your virtual environment active, installing Django is easy using pip.

  1. Install Django: Run the following command:
    pip install django
    pip will download and install the latest stable version of Django and its dependencies within your active virtual environment. You'll see output showing the download and installation progress.

Step 5: Verify Django Installation

Let's quickly check if Django was installed correctly.

  1. Check Django Version: With the virtual environment still active, run:
    django-admin --version
    You should see the installed Django version number (e.g., 4.2.3).

Step 6: Create Your First Django Project

Django comes with command-line tools to help you start projects and apps. Let's create the basic project structure.

  1. Create Project: Make sure you are still in the mydjangoproject directory (where your venv folder is) and your virtual environment is active. Run:

    django-admin startproject myproject .
    • django-admin startproject: The command to create a new project.
    • myproject: The name you want to give your Django project configuration folder.
    • . (dot): This is important! It tells Django to create the project files in the current directory (mydjangoproject) rather than creating an extra nested folder.

    After running this, you'll see a new folder named myproject (containing settings, URLs, etc.) and a file named manage.py appear alongside your venv folder. manage.py is your command-line utility for interacting with your Django project.

Step 7: Run the Development Server

Let's see if it works! Django includes a lightweight web server for development purposes.

  1. Run Server: Make sure your virtual environment is active and you are in the directory containing manage.py. Run:
    python manage.py runserver
  2. Check Output: You'll see some output, including lines like:
    Starting development server at http://127.0.0.1:8000/
Quit the server with CTRL-BREAK.
  3. Visit in Browser: Open your web browser and navigate to http://127.0.0.1:8000/ (or whatever address is shown in your terminal).

If everything went well, you should see the default Django success page with a rocket – Congratulations!

  1. Stop the Server: Go back to your Command Prompt and press Ctrl + C to stop the development server.

Conclusion

That's it! You've successfully installed Python, set up a virtual environment, installed Django, and verified it by running the development server on your Windows machine.

You're now ready to follow the official Django tutorial or other resources to start building your web application. Remember to activate your virtual environment (.\venv\Scripts\activate) every time you start working on your project.

Happy coding!

Key Takeaways:

  • Always use virtual environments (venv) for your Python projects.
  • Remember to activate the environment (source <env_name>/bin/activate) before working on a project.
  • Use pip install django inside the activated environment.
  • django-admin startproject <name> . creates your project structure.
  • python manage.py runserver starts the development server.

You're now ready to follow the official Django tutorial or start building your own web applications. Happy coding!
