How to Create Django Templates

[MISSING TRANSLATION] aymen March 29, 2025

So, you're building a web application with Django. You've got your models handling data and your views processing requests, but how do you actually show something dynamic and well-structured to your users in their browser? The answer lies in Django Templates.

Templates are the presentation layer in Django's Model-View-Template (MTV) architecture. They allow you to separate the design and layout (HTML, CSS, JavaScript) from your Python logic (views). This separation is crucial for maintainable, scalable, and collaborative web development.

Let's dive into how you can create and use them effectively.

What Are Django Templates?

At their core, Django templates are text files (usually HTML) that contain a mix of static content and special placeholders for dynamic data. Django's template engine processes these files, replacing the placeholders with actual values provided by your views and rendering the final HTML to be sent to the user's browser.

Think of them like mail merge documents: you have a standard letter (the template) with spots for the recipient's name and address (dynamic data).

Why Use Templates?

  1. Separation of Concerns: Keeps your Python code focused on logic and your HTML focused on presentation. Clean code is happy code!
  2. Reusability: Create base templates and include snippets to avoid repeating common HTML structures (like headers, footers, navbars) across multiple pages (DRY - Don't Repeat Yourself).
  3. Collaboration: Front-end developers or designers can work on the HTML/CSS/JS in the template files without needing deep knowledge of the backend Python code.
  4. Power & Flexibility: The Django Template Language (DTL) provides tags and filters to handle basic logic, loops, conditionals, and data formatting directly within the template.

Setting Up Your Templates Directory

First, Django needs to know where to find your template files.

  1. Inside your App: You can create a templates directory inside each of your Django apps. Inside that directory, create another directory named after your app. Example: my_app/templates/my_app/my_template.html. This namespacing prevents clashes if multiple apps have templates with the same name. Django automatically finds these if APP_DIRS is set to True in your settings.py (which it is by default).

  2. Project-Level Templates: For templates shared across multiple apps (like a base layout), you can define a project-level templates directory.

    • Create a templates folder in your project's root directory (alongside manage.py).
    • Tell Django about it in your settings.py:
    # settings.py
import os

BASE_DIR = os.path.dirname(os.path.dirname(os.path.abspath(__file__))) # Or Path(__file__).resolve().parent.parent for pathlib users

TEMPLATES = [
    {
        'BACKEND': 'django.template.backends.django.DjangoTemplates',
        # Add your project-level templates directory here
        'DIRS': [os.path.join(BASE_DIR, 'templates')],
        'APP_DIRS': True, # Tells Django to look inside app directories too
        'OPTIONS': {
            'context_processors': [
                'django.template.context_processors.debug',
                'django.template.context_processors.request',
                'django.contrib.auth.context_processors.auth',
                'django.contrib.messages.context_processors.messages',
            ],
        },
    },
]

Creating Your First Template

Let's create a simple template. Inside your project-level templates directory (or my_app/templates/my_app/), create a file named hello.html:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Greetings!</title>
</head>
<body>
    <h1>Hello, {{ name }}!</h1>
    <p>Welcome to our Django site.</p>
</body>
</html>

Notice the {{ name }} part? That's a template variable. Django will replace this with a value we provide from our view.

Rendering the Template from a View

Now, let's create a view in your app's views.py file to render this template.

# my_app/views.py
from django.shortcuts import render

def say_hello(request):
    # Data you want to pass to the template
    context = {
        'name': 'Django Learner'
    }
    # Render the template with the context data
    return render(request, 'hello.html', context) # Or 'my_app/hello.html' if namespaced

Explanation:

  1. We import the render shortcut.
  2. We define a view function say_hello.
  3. We create a dictionary called context. This dictionary holds the key-value pairs that will be available as variables in our template. Here, the key 'name' corresponds to {{ name }} in the template.
  4. render(request, 'template_name.html', context) does the magic: it loads the specified template, fills in the placeholders using the context dictionary, and returns an HttpResponse object containing the rendered HTML.

Finally, wire up this view to a URL in your urls.py:

# my_project/urls.py or my_app/urls.py
from django.urls import path
from my_app import views # Assuming your view is in my_app

urlpatterns = [
    path('hello/', views.say_hello, name='hello'),
    # ... other urls
]

Now, if you run your development server (python manage.py runserver) and navigate to /hello/, you should see "Hello, Django Learner!".

The Power of the Django Template Language (DTL)

DTL offers more than just variables:

  1. Tags ({% tag %}): Control logic and flow.

    • {% if condition %} ... {% else %} ... {% endif %}: Conditional rendering.
    • {% for item in item_list %} ... {% empty %} ... {% endfor %}: Looping over lists or querysets. {% empty %} is shown if the list is empty.
    • {% url 'url_name' %}: Generates URLs dynamically based on the name defined in urls.py. (Highly recommended!)
    • {% static 'path/to/file.css' %}: Generates URLs for static files (CSS, JS, images). Requires {% load static %} at the top of the template.
    • {% extends 'base.html' %} & {% block content %}: Template inheritance (see below).
    • {% include 'snippet.html' %}: Includes another template file.

  2. Filters ({{ variable|filter }}): Modify how variables are displayed.

    • {{ name|lower }}: Converts the value to lowercase.
    • {{ my_list|length }}: Returns the length of the list.
    • {{ description|truncatewords:15 }}: Truncates text to a specific number of words.
    • {{ pub_date|date:"F j, Y" }}: Formats a date object.
    • {{ value|default:"Nothing" }}: Provides a default value if the variable is false or empty.

Example using Tags and Filters:

<!-- products.html -->
{% load static %} <!DOCTYPE html>
<html>
<head>
    <title>Our Products</title>
    <link rel="stylesheet" href="{% static 'css/style.css' %}">
</head>
<body>
    <h1>Available Products ({{ products|length }})</h1>

    {% if products %}
        <ul>
            {% for product in products %}
                <li>
                    {{ product.name|capfirst }} - ${{ product.price|floatformat:2 }}
                    {% if product.is_on_sale %}
                        <span style="color: red;">(SALE!)</span>
                    {% endif %}
                </li>
            {% endfor %}
        </ul>
    {% else %}
        <p>Sorry, no products available right now.</p>
    {% endif %}

    <p><a href="{% url 'home' %}">Back to Home</a></p>
</body>
</html>

Template Inheritance: Staying DRY

This is one of the most powerful features. Instead of repeating your entire HTML structure (header, footer, sidebar) on every page, you create a base template and have child templates "extend" it.

1. Create base.html (e.g., in your project templates folder):

<!-- templates/base.html -->
{% load static %}
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>{% block title %}My Awesome Site{% endblock %}</title>
    <link rel="stylesheet" href="{% static 'css/base.css' %}">
    {% block extra_head %}{% endblock %}
</head>
<body>
    <nav>
        <!-- Your site navigation -->
        <a href="{% url 'home' %}">Home</a> |
        <a href="{% url 'about' %}">About</a>
    </nav>

    <main>
        {% block content %}
            <!-- Default content goes here if child doesn't provide it -->
        {% endblock %}
    </main>

    <footer>
        <p>&copy; {% now "Y" %} My Awesome Site. All rights reserved.</p>
    </footer>

    {% block extra_scripts %}{% endblock %}
</body>
</html>

Key points:

  • {% block block_name %} ... {% endblock %} defines sections that child templates can override.
  • We included blocks for the page title, main content, extra CSS/JS (extra_head, extra_scripts).

2. Create a Child Template (e.g., about.html):

<!-- my_app/templates/my_app/about.html -->
{% extends 'base.html' %}
{% load static %}

{% block title %}About Us - {{ block.super }}{% endblock %}

{% block extra_head %}
    <link rel="stylesheet" href="{% static 'css/about_page.css' %}">
{% endblock %}

{% block content %}
    <h1>About Our Company</h1>
    <p>This is where we talk about how great we are!</p>
    <p>Inheritance makes template management much easier.</p>
{% endblock %}

Key points:

  • {% extends 'base.html' %} MUST be the very first tag.
  • We override the title and content blocks.
  • {{ block.super }} in the title block includes the original content from the parent block ("My Awesome Site").
  • We also added specific CSS for this page using the extra_head block.

Now, when you render about.html from a view, Django will automatically pull in the structure from base.html and insert the content from the blocks defined in about.html.

Conclusion

Django templates are fundamental to building user interfaces for your web applications. By understanding how to configure them, pass data from views using context, leverage the Django Template Language (DTL) with its variables, tags, and filters, and master template inheritance, you can create dynamic, maintainable, and professional-looking websites.

Start simple, experiment with tags and filters, and embrace the power of inheritance. Happy templating!
