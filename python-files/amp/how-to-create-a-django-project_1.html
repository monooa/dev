How to create a Django project

[MISSING TRANSLATION] aymen March 29, 2025

So, you've heard about Django, the high-level Python web framework that encourages rapid development and clean, pragmatic design. You're excited to build something awesome, but first things first: how do you actually start a Django project?

Don't worry, it's simpler than you might think! This guide will walk you through the essential steps to get your first Django project up and running.

What is Django?

Briefly, Django is a "batteries-included" framework. This means it comes with many tools you'll need right out of the box, like an Object-Relational Mapper (ORM) for database interactions, an authentication system, a templating engine, and an admin interface. It follows the Model-View-Template (MVT) architectural pattern.

Prerequisites

Before we dive in, make sure you have the following installed:

  1. Python: Django is a Python framework, so you need Python installed. You can download it from python.org. Django requires a specific Python version; check the Django documentation for compatibility with the version you want to use. Open your terminal or command prompt and type python --version or python3 --version to check.
  2. pip: This is the Python package installer. It usually comes bundled with modern Python versions. You can check with pip --version or pip3 --version.

Let's Get Building!

Step 1: Create and Activate a Virtual Environment (Highly Recommended!)

Why use a virtual environment? It creates an isolated space for your project's dependencies. This means packages you install for this project won't interfere with other Python projects on your system, and vice-versa. It's crucial for good project management.

  • Navigate to the directory where you want to store your project using your terminal or command prompt.

  • Create the virtual environment:

    # On macOS/Linux
python3 -m venv venv

# On Windows
python -m venv venv

    (This creates a directory named venv – you can name it differently, but venv is conventional).

  • Activate the virtual environment:

    # On macOS/Linux
source venv/bin/activate

# On Windows (Command Prompt)
.\venv\Scripts\activate

# On Windows (PowerShell or Git Bash)
source venv/Scripts/activate

    You'll know it's active because your command prompt will usually be prefixed with (venv).

Step 2: Install Django

Now that your virtual environment is active, you can install Django safely within it.

pip install django

Pip will download and install the latest stable version of Django and its dependencies. You can verify the installation:

django-admin --version

Step 3: Create Your Django Project

This is where the magic starts! Django provides a command-line utility called django-admin to handle various tasks, including creating a new project.

  • Make sure you are outside the venv directory but still in the folder where you want your project code to live.
  • Run the startproject command, replacing myprojectname with your desired project name (use letters, numbers, and underscores only, and don't use hyphens or names like django or test):
django-admin startproject myprojectname .
  • Wait, what's the dot (.) at the end? This tells Django to create the project files directly in the current directory. If you omit the dot (django-admin startproject myprojectname), Django will create a new directory called myprojectname and put the project files inside that. Using the dot often results in a slightly cleaner top-level structure, but either way works!

Step 4: Explore the Project Structure

If you used the . command, your current directory will now contain:

  • manage.py: A command-line utility specific to this project. You'll use it constantly for tasks like running the development server, creating database migrations, starting new apps, etc. Think of it as your project's command center.
  • myprojectname/ (a directory with the same name as your project): This is the actual Python package for your project.
    • __init__.py: An empty file that tells Python this directory should be considered a Python package.
    • settings.py: Contains all the configuration for your Django project (database settings, installed apps, middleware, static files, template directories, secret key, etc.). You'll edit this file often.
    • urls.py: The URL declarations for your project; a "table of contents" for your site. Defines which Python functions (views) should be called for specific URL patterns.
    • asgi.py: An entry-point for ASGI-compatible web servers (for asynchronous features).
    • wsgi.py: An entry-point for WSGI-compatible web servers (the standard for synchronous Python web apps).

Step 5: Run the Development Server

Let's see if it works! Django comes with a lightweight web server for development purposes.

  • Make sure you are in the directory containing manage.py.
  • Run the following command:
python manage.py runserver

You should see output similar to this in your terminal:

Watching for file changes with StatReloader
Performing system checks...

System check identified no issues (0 silenced).

You have unapplied migrations; your app may not work properly until they are applied.
Run 'python manage.py migrate' to apply them.

September 05, 2023 - 10:30:00
Django version 4.x.x, using settings 'myprojectname.settings'
Starting development server at http://127.0.0.1:8000/
Quit the server with CONTROL-C.
  • Open your web browser and navigate to http://127.0.0.1:8000/ (or the address shown in your terminal).
  • You should see the default Django welcome page, often featuring a rocket taking off! Congratulations, your Django project is running!
  • You can stop the development server by pressing Ctrl+C in the terminal where it's running.

Step 6: Apply Initial Migrations

You might have noticed a message about "unapplied migrations" when running the server. Django comes with built-in apps (like authentication, admin, sessions) that require database tables. Migrations are Django's way of propagating changes you make to your models (database schema) into the database itself.

Let's apply the initial migrations for the built-in apps.

python manage.py migrate

You'll see output showing Django creating the necessary tables in a default SQLite database file (db.sqlite3) that it creates automatically in your project directory.

What's Next?

You've successfully created and run your first Django project! This is the foundation. From here, you'll typically:

  1. Create an App: Django projects are made up of "apps." Each app handles a specific piece of functionality (e.g., a blog app, a user profiles app, a polls app). Use python manage.py startapp myappname.
  2. Define Models: Design your database structure in models.py within your app.
  3. Create Migrations: Run python manage.py makemigrations and python manage.py migrate after changing models.
  4. Write Views: Create Python functions or classes in views.py to handle requests and return responses.
  5. Define URLs: Map URL patterns to your views in your app's urls.py and include it in the main project's urls.py.
  6. Build Templates: Create HTML files (.html) in a templates directory to define the structure of your web pages.

Starting a new project is the first exciting step into the powerful world of Django. Keep exploring, keep building, and don't hesitate to consult the excellent official Django documentation! Happy coding!
