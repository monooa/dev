How to install Django on Linux

aymen March 29, 2025

So, you want to build powerful web applications, and you've heard great things about Django? Excellent choice! Django is a high-level Python web framework that encourages rapid development and clean, pragmatic design. And what better place to run it than on a robust Linux system?

Whether you're using Ubuntu, Fedora, Debian, or another popular distribution, installing Django is a straightforward process. This guide will walk you through the essential steps, emphasizing best practices like using virtual environments.

Let's get started!

Prerequisites:

Before we dive into Django, you'll need a couple of things installed on your Linux system:

  1. Python 3: Django requires Python. Most modern Linux distributions come with Python 3 pre-installed.
  2. pip: The Python package installer. This is often installed alongside Python 3, but sometimes needs to be installed separately.
  3. (Optional but Recommended) python3-venv: The module for creating virtual environments.

Step 1: Check Your Python and pip Installation

First, let's verify if you have Python 3 and pip installed and check their versions. Open your terminal and run:

python3 --version
pip3 --version

If these commands return version numbers (e.g., Python 3.8.10, pip 20.0.2), you're good to go!

What if Python 3 or pip isn't installed?

No worries! You can easily install them using your distribution's package manager.

  • On Debian/Ubuntu:

    sudo apt update
sudo apt install python3 python3-pip python3-venv

  • On Fedora/CentOS/RHEL:

    sudo dnf update  # or 'sudo yum update' on older CentOS/RHEL
sudo dnf install python3 python3-pip python3-venv # or 'sudo yum install ...'

(Remember: Use sudo only for system-level package installations, not for installing Python packages with pip later on.)

Step 2: Create and Activate a Virtual Environment (Highly Recommended!)

Why use a virtual environment? Imagine you're working on multiple Python projects. Project A might need Django version 4.0, while Project B needs Django 3.2. A virtual environment creates an isolated space for each project, keeping their dependencies separate and preventing conflicts. It's a crucial best practice in Python development.

  1. Navigate to your project directory: Go to where you want to create your Django project.

    mkdir my_django_project
cd my_django_project

  2. Create the virtual environment: We'll use Python's built-in venv module. Let's name our environment env (a common convention).

    python3 -m venv env

    This command creates an env/ directory containing a copy of Python and pip specific to this project.

  3. Activate the virtual environment: Before installing packages for your project, you need to activate the environment.

    source env/bin/activate

    You'll know it's active because your terminal prompt will change, usually prepending (env) like this:

    (env) user@hostname:~/my_django_project$

    Now, any pip install commands will install packages inside this isolated env directory, not globally on your system.

Step 3: Install Django

With your virtual environment activated, installing Django is incredibly simple using pip:

pip install Django

Pip will download and install the latest stable version of Django and its dependencies within your active virtual environment.

Step 4: Verify the Installation

Let's quickly check if Django was installed correctly. Run this command:

django-admin --version

If Django is installed properly, this will print its version number (e.g., 4.1.7).

Congratulations! You've successfully installed Django inside a dedicated virtual environment on your Linux machine.

What's Next?

You're now ready to start building! The next logical step is to create your first Django project:

# Make sure you are still in your project directory (e.g., my_django_project)
# and your virtual environment (env) is active.
django-admin startproject mywebapp .

(Note the . at the end – this tells Django to create the project files in the current directory rather than creating an extra subdirectory.)

This command will set up the basic directory structure and files for your new Django web application named mywebapp. From here, you can run the development server (python manage.py runserver) and start building your views, models, and templates.

Deactivating the Virtual Environment

When you're finished working on your project, you can deactivate the virtual environment by simply typing:

deactivate

Your prompt will return to normal. Remember to reactivate it (source env/bin/activate) the next time you work on this project.

Happy Djangonauting on Linux! Feel free to leave comments below if you encounter any issues or have questions.
