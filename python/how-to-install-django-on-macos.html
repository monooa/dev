How to install Django on macOS

aymen March 29, 2025

So, you're ready to dive into the world of web development with Django, the high-level Python web framework that encourages rapid development and clean, pragmatic design. You're using a Mac, and you want to get set up. Great choice! macOS is a fantastic environment for Python and Django development.

This guide will walk you through the process step-by-step, ensuring you have a clean and isolated development environment.

What We'll Cover:

  1. Prerequisites: Checking for Python and Pip.
  2. Virtual Environments: Why they're essential and how to create one.
  3. Installing Django: Getting the framework itself.
  4. Verification: Making sure everything worked.
  5. Bonus: Creating your very first Django project.

Let's get started!

Step 1: Check Your Prerequisites (Python & Pip)

Django is a Python framework, so you need Python installed. Modern macOS versions usually come with Python 3 pre-installed, which is what Django requires. You also need pip, the Python package installer, which typically comes bundled with Python 3.

Let's verify:

  1. Open Terminal: You can find this in Applications -> Utilities -> Terminal, or simply search for "Terminal" using Spotlight (Cmd + Space).

  2. Check Python Version: Type the following command and press Enter:

    python3 --version

    You should see output like Python 3.x.y. If you see an error or a version older than Python 3.6 (though the latest Python 3 is recommended), you'll need to install or update Python 3. You can download it from the official Python website or install it using a package manager like Homebrew (brew install python).

  3. Check Pip Version: Pip is usually installed alongside Python 3. Check its version:

    pip3 --version

    You should see output indicating the pip version and its location. If it's missing, you might need to reinstall Python 3. It's also good practice to ensure pip is up-to-date:

    python3 -m pip install --upgrade pip

With Python 3 and pip ready, you're set for the next crucial step.

Step 2: Create a Virtual Environment

Why use a virtual environment? Imagine working on multiple Python projects. Project A might need Django version 3.2, while Project B needs Django 4.1. If you install packages globally (directly onto your system), you'll run into conflicts.

A virtual environment creates an isolated space for each project. Packages installed within it don't affect other projects or your system-wide Python installation. It's standard practice and highly recommended!

  1. Choose a Project Directory: Navigate in your Terminal to where you want to keep your Django projects. You can create a new directory if needed:
    # Example: Create and navigate to a 'django_projects' directory in your home folder
mkdir ~/django_projects
cd ~/django_projects
  2. Create the Virtual Environment: We'll use venv, Python's built-in module. Replace my_django_env with your preferred environment name (using venv or .venv is also common).
    python3 -m venv my_django_env
    This command creates a folder named my_django_env containing a copy of the Python interpreter and a place to install project-specific libraries.

Step 3: Activate the Virtual Environment

Before installing Django, you need to "activate" the environment you just created.

  1. Activate: Run the following command (make sure you're in the directory containing my_django_env):
    source my_django_env/bin/activate
  2. Verify Activation: You'll know it's active because your Terminal prompt will change, usually prepending the environment name in parentheses, like this:
    (my_django_env) your_username@your_mac ~ %
    Important: Every time you want to work on this specific project, you need to navigate to its directory and run the source my_django_env/bin/activate command first. When you're done, simply type deactivate to exit the virtual environment.

Step 4: Install Django

Now that your virtual environment is active, installing Django is simple. Pip will install it inside this isolated environment.

pip install django

Pip will download and install the latest stable version of Django and its dependencies. You'll see output indicating the progress and successful installation.

Step 5: Verify Django Installation

Let's quickly check if Django was installed correctly.

django-admin --version

Alternatively, you can use:

python -m django --version

Either command should print the installed Django version (e.g., 4.2.7). Success!

Bonus Step: Create Your First Django Project

With Django installed, the next logical step is to create a project skeleton.

  1. Create Project: Make sure your virtual environment is still active. Run the django-admin startproject command. Replace myproject with your desired project name. The . at the end tells Django to create the project files in the current directory (recommended to avoid an extra nested folder).

    django-admin startproject myproject .

    This command creates several files and a subdirectory, including manage.py (a command-line utility for interacting with your project) and a folder named myproject containing project-level settings.

  2. Run the Development Server: Let's make sure the basic project works.

    python manage.py runserver

    You should see output like this:

    Watching for file changes with StatReloader
Performing system checks...

System check identified no issues (0 silenced).
September 28, 2023 - 10:00:00
Django version 4.x.y, using settings 'myproject.settings'
Starting development server at http://127.0.0.1:8000/
Quit the server with CONTROL-C.

  3. Visit in Browser: Open your web browser and go to http://127.0.0.1:8000/. You should see the default Django welcome page ("The install worked successfully! Congratulations!").

  4. Stop the Server: Go back to your Terminal and press Ctrl + C to stop the development server.

Conclusion

Congratulations! You've successfully installed Django on your macOS system within a clean virtual environment and even verified it by creating and running a basic project.

Key Takeaways:

  • Always use virtual environments (venv) for your Python projects.
  • Remember to activate the environment (source <env_name>/bin/activate) before working on a project.
  • Use pip install django inside the activated environment.
  • django-admin startproject <name> . creates your project structure.
  • python manage.py runserver starts the development server.

You're now ready to follow the official Django tutorial or start building your own web applications. Happy coding!
